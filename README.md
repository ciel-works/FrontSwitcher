# FrontSwitcher

Windows 常駐型のウインドウ操作ツール。
ショートカットキー1つで、作業中（前面）のウインドウを最小化します。もう一度押すと元に戻ります（トグル）。
オプションで「指定したアプリを最前面に呼び出す」動作も追加できます。

## 主な機能

- **タスクトレイ常駐** — 本体はトレイアイコンに格納。バックグラウンドで動作。
- **グローバルホットキー** — 任意のアプリで使用中でも、登録したキーで切替。
- **最小化のみ（既定）** — ホットキーで前面ウインドウを最小化。画面の動きが小さい。
- **対象アプリの最前面化**（オプション・既定 OFF） — ON にすると最小化に加えて指定アプリを最前面へ。未起動なら設定した exe を自動起動。
- **一緒に最小化するアプリ**（オプション・最大20） — 事前に登録したアプリが開いていれば、前面ウインドウと一緒に最小化（再押下でまとめて復元）。両モードで有効。
- **タブを閉じる**（オプション・最大20） — Chrome / Edge で、タブのタイトルが登録した正規表現にマッチしたら、ホットキー時に一緒に閉じる（UI Automation 利用）。⚠️ 閉じたタブは元に戻せません（Ctrl+Shift+T で手動復元は可）。
- **トグル動作** — 同じキーの再押下で、最小化した元ウインドウ（と一緒に最小化したアプリ）を復元。
- **タスクバーから隠す**（任意） — 最小化した作業中ウインドウをタスクバーから消す。既定 OFF。
  - ⚠️ Windows 11 ではタスクバーと Alt+Tab が連動するため、**Alt+Tab からも消えます**。戻すときは「同じホットキーをもう一度押す」か、トレイメニューの「**隠したウインドウを戻す**」を使ってください（タスクバーのボタンも復活します）。
- **簡易設定画面** — ホットキー・対象アプリ・exe パス・各種オプションを設定。
- **管理者権限で動作** — 管理者権限で動く他アプリ（DRM ビューア等）のウインドウも最小化できるよう、常に管理者として起動します（マニフェスト `requireAdministrator`）。手動起動時は UAC 確認が出ます。
- **Windows スタートアップ登録（UACなし）**（任意） — タスク スケジューラの「最上位の特権で実行」＋ログオン時トリガーで登録。ログオン時に UAC を出さずに管理者起動します（※管理者アカウントが必要）。

## 設定の保存場所

```
%AppData%\FrontSwitcher\settings.json
```

## 動作要件

- Windows 10 / 11
- .NET 8 Desktop Runtime（インストーラ配布時は同梱 or 自己完結発行を推奨）

## ビルド

```powershell
cd C:\Users\森川組総務部\source\FrontSwitcher
dotnet build -c Release
```

実行ファイル: `bin\Release\net8.0-windows\FrontSwitcher.exe`

### 配布用（自己完結・単一ファイル）

ランタイム未導入の PC でも動くように発行する場合:

```powershell
dotnet publish -c Release -r win-x64 --self-contained true `
  -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true
```

出力: `bin\Release\net8.0-windows\win-x64\publish\FrontSwitcher.exe`

## 使い方

1. `FrontSwitcher.exe` を起動（トレイに常駐）。
2. トレイアイコンを右クリック →「設定...」。
3. **ショートカットキー** 欄をクリックし、登録したいキーの組み合わせを押す（既定: `Ctrl + Alt + Z`）。
4. **対象アプリ** を指定:
   - プロセス名を直接入力（例: `notepad`, `chrome`, `EXCEL`）。
   - または「前面ウインドウから取得」で、3秒以内に対象アプリを前面にして自動取得。
   - 未起動時に自動起動させたい場合は **exe パス** も設定（「参照...」）。
5. 「保存」。
6. 任意のアプリ作業中にホットキーを押すと切替。もう一度で復帰。

## トラブルシューティング

- **ホットキーが効かない** → 他アプリと競合の可能性。設定で別のキーに変更。
- **対象が前面に来ない** → プロセス名が正しいか確認（タスクマネージャの「詳細」タブの名前から `.exe` を除いたもの）。
- **トレイアイコンが出ない** → タスクバーの「隠れているインジケーター」を確認。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `App.xaml(.cs)` | エントリポイント・トレイ常駐・全体制御 |
| `SettingsWindow.xaml(.cs)` | 設定画面 |
| `AppSettings.cs` | 設定モデル・JSON 永続化 |
| `HotKeyService.cs` | グローバルホットキー登録 |
| `WindowSwitcher.cs` | ウインドウ切替ロジック |
| `NativeMethods.cs` | Win32 API ラッパー |
| `StartupManager.cs` | スタートアップ登録 |
| `HotKeyText.cs` | ホットキー表記・変換 |
